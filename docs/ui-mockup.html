05 — design notes · 设计原则

编辑式排版

不用 card,只用 column 与 rule。 所有分隔靠 1 像素的发丝线,所有信息按基线对齐。 正文采用 Geist + Noto Sans SC,标题与温度数字用 Instrument Serif (含 Noto Serif SC 中文回退), 技术数值统一用 JetBrains Mono 等宽体,让温度、湿度、风速形成可扫读的"数据栏"。

温度即英雄

温度数字采用 240 px(桌面)/ 152 px(移动)细衬线, 让一个数字承担页面的视觉重量。 度数符号缩小并上移,避免和数字争抢空间。 所有数字 tabular-nums 等宽对齐, 切换 °C / °F 时不抖动。

背景即天况

七种主题 (晴日/晴夜/多云/雨/雪/雷暴/雾) 各有专属的多层径向渐变 + 实物 SVG (太阳、月亮、云、闪电、山线) + CSS 动画 (雨滴下落、雪花飘移、星辰闪烁、雷光闪烁、雾带横移)。 不使用 backdrop-filter 堆砌"磨砂玻璃", 所有视觉效果可在 60fps 下渲染。

收藏即页脚

收藏栏不占主区域,以底部一行 inline list 呈现, 城市以中文衬线标识,温度以等宽数字尾随, hover 显示删除按钮。 不喧宾夺主,但又便于点击切换。

日轨叙事

用一条从左到右的弧线代替"日出 — 日落"两个孤立时间戳, 在弧上以一个发光小点指示当前时刻位置。 时间因此被空间化,用户一眼看到"现在到日落还多远"。

实现可行

设计为 React + TS + Tailwind 而生。 所有动效是纯 CSS,所有图形是 inline SVG, 所有主题切换通过 data-theme 属性 + CSS 变量, 0 第三方动画/插件依赖。 暗色与亮色由场景自身决定,无需另设主题切换。
